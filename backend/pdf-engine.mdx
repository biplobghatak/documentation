title: "PDF Engine"
description: "How Spyro turns reports into downloadable PDFs - an external Gotenberg service renders the shared branded report (free tools + the in-app audit), with react-pdf as a legacy fallback for older audits."
---

Spyro generates PDFs through **two render mechanisms**, chosen per report:

| Mechanism | Library | Used by | How it renders |
| --- | --- | --- | --- |
| **Gotenberg** (primary) | external headless-Chrome service | the **five free tools** _and_ the authed **Site Audit** export (any audit with a stored V3 report) | POSTs a public HTML page's URL to Chromium and streams the PDF back |
| **react-pdf** (legacy fallback) | `@react-pdf/renderer` | only **older audits** with no stored `reportJson` (run before migration `0063`) | builds a PDF document tree in Node, no browser |

The important shift from earlier versions: the authed Site Audit no longer
renders with react-pdf. Modern audits store a full **V3 SEO+GEO report** on the
row (`audits.reportJson`, see [Audit](/backend/audit)), and the export renders
that through Gotenberg using the **same branded document as the free SEO/GEO
tool**. react-pdf survives only as a fallback for audits created before that
column existed.

<Note>
There is **no Chromium binary in the Vercel bundle**. The old
`puppeteer-core`/`@sparticuz/chromium` approach was removed - PDFs are rendered
by the external Gotenberg service instead. See the comment in
`next.config.ts:34-36`.
</Note>

## The Gotenberg path (the default)

Both the [free tools](/backend/free-tools) and modern audits produce full
HTML/CSS report pages, not react-pdf trees. To turn one into a pixel-perfect
PDF, Spyro renders it with **real Chromium** running in an external
[Gotenberg](https://gotenberg.dev) container (self-hosted on Railway). The Vercel
function never touches a browser binary - it just calls Gotenberg over HTTP.

### The render helper

All PDF routes funnel through one helper, `renderReportPdf`, in
`lib/free-tools/shared/pdf/browser.ts`:

```ts
// lib/free-tools/shared/pdf/browser.ts:3-9
/**
 * PDF engine: Gotenberg (self-hosted headless Chrome on Railway).
 *
 * The report pages are full HTML/CSS pages. We render them with real Chromium
 * by POSTing the report's PUBLIC URL to a Gotenberg container and streaming back
 * the PDF, so the Vercel function carries no browser binary.
 */
```

It builds a public URL to the report page, POSTs it as multipart form data to
Gotenberg's URL-conversion endpoint, and streams the result back:

```ts
// lib/free-tools/shared/pdf/browser.ts:92-115
const form = new FormData();
form.append("url", reportUrl);
form.append("paperWidth", "8.27");   // A4
form.append("paperHeight", "11.69");
form.append("marginTop", "0.55");
// … margins + printBackground …
form.append("waitDelay", "1s");      // let Inter/Lato web fonts settle

res = await fetch(`${GOTENBERG_URL}/forms/chromium/convert/url`, {
  method: "POST",
  body: form,
  headers: authHeaders(),
  signal: controller.signal,
});
```

<Warning>
Gotenberg fetches the report page **itself**, over the public internet - it
cannot reach `localhost`. The origin it fetches is resolved by
`resolveReportOrigin` (`browser.ts:42-59`), which tries, in order:
`GOTENBERG_REPORT_ORIGIN` → `NEXT_PUBLIC_APP_URL` →
`VERCEL_PROJECT_PRODUCTION_URL` → `VERCEL_URL` → the request's own host, and
**rejects any localhost/127.0.0.1 candidate**. For local dev you must point
`GOTENBERG_REPORT_ORIGIN` at a public origin (e.g. the prod domain, which reads
the same shared DB).
</Warning>

### Environment variables

All Gotenberg config is read at the top of `browser.ts` (`:19-21`,
documented in `:10-17`):

| Variable | Purpose |
| --- | --- |
| `GOTENBERG_URL` | Base URL of the Gotenberg service (required; trailing slashes stripped). |
| `GOTENBERG_USER` / `GOTENBERG_PASSWORD` | Optional HTTP basic-auth credentials, sent as a `Basic` header when both are set (`browser.ts:27-33`). |
| `GOTENBERG_REPORT_ORIGIN` | Optional override for the public origin Gotenberg fetches the report from. Must be publicly reachable. |

See [Environment variables](/reference/environment-variables) for the full list.

### The shared report document

Both render pages delegate their entire layout to one component,
`components/audit/audit-report-document.tsx` - `AuditReportDocument`. It is a
branded, print-ready Tailwind document (Lato headings + Inter body, the site's
own fonts; brand `#ea580c`; CSS `conic-gradient` score gauges and `@media print`
rules). Every value is read straight from the `report` prop - **nothing is
recomputed in the document** (`audit-report-document.tsx:17-29`):

```tsx
// components/audit/audit-report-document.tsx - one document, two variants
export function AuditReportDocument({
  report,
  variant,           // "free" | "app"
}: { report: FreeAuditReport; variant: "free" | "app" }) { … }
```

The **only** difference between the two variants is the closing CTA section
(`variant="free"` nudges signup; `variant="app"` is the in-product export). This
is why the free tool and the in-app audit produce visually identical PDFs.

### The two render pages

The HTML that Gotenberg prints is an ordinary Next.js **server component** that
reads a stored report from the database and hands it to `AuditReportDocument` -
no re-crawl:

<CardGroup cols={2}>
  <Card title="/pdf-report (free tools)" icon="globe">
    `app/pdf-report/page.tsx` - **public**, keyed on the `?website=` domain. Reads
    the cached `free_audit_reports` row via `getReportRow` + `isFresh`, then
    renders `<AuditReportDocument variant="free" />` (`pdf-report/page.tsx:14-32`).
  </Card>
  <Card title="/audit-report (in-app audit)" icon="lock">
    `app/audit-report/page.tsx` - **token-gated**. Reads `audits.report_json` for
    the `?auditId=`, but only after `verifyAuditPdfToken(auditId, token)` passes,
    then renders `<AuditReportDocument variant="app" />` (`audit-report/page.tsx:20-39`).
  </Card>
</CardGroup>

```tsx
// app/audit-report/page.tsx:25-38 - anonymous, HMAC-token-gated
const { auditId, token } = await searchParams;
if (!auditId || !token || !verifyAuditPdfToken(auditId, token)) return notFound();

const row = await db.select({ reportJson: audits.reportJson })
  .from(audits).where(eq(audits.id, auditId)).limit(1).then((r) => r[0] ?? null);
if (!row?.reportJson) return notFound();

const report = normalizeReport(row.reportJson);
return <AuditReportDocument report={report} variant="app" />;
```

<Warning>
`/audit-report` is in `proxy.ts`'s public allowlist so Gotenberg (which has no
user session) can fetch it - see [Middleware](/backend/middleware). It is **not**
unprotected: it requires a short-lived HMAC token that only the authenticated PDF
route mints. Without a valid, unexpired token it 404s.
</Warning>

## The audit PDF download route

`app/api/audit/[auditId]/pdf/route.ts` is the `GET` handler for the authed
export. It is **auth-gated and owner-scoped** (a private report), then branches
on whether a V3 report was stored:

```ts
// app/api/audit/[auditId]/pdf/route.ts:38-81
const user = await getCurrentUser();
if (!user) return new NextResponse("Unauthorized", { status: 401 });

// Audit scoped to this user - prevents cross-user access.
const auditRow = await db.select().from(audits)
  .where(and(eq(audits.id, auditId), eq(audits.userId, user.id)))
  .limit(1).then((r) => r[0] ?? null);
if (!auditRow || auditRow.status !== "done") return new NextResponse("Not found", { status: 404 });

// Modern path: a full V3 report was stored at audit time → render the rich
// branded report via Gotenberg, authorized by a short-lived HMAC token minted
// here (after the auth + ownership check above).
if (auditRow.reportJson) {
  const token = signAuditPdfToken(auditId);
  return renderReportPdf({
    req,
    tool: "audit",
    reportPath: "/audit-report",
    query: `auditId=${encodeURIComponent(auditId)}&token=${encodeURIComponent(token)}`,
    filename,
  });
}
// else → legacy react-pdf fallback (below)
```

So the route does the privileged work - verify the signed-in user owns the audit - **once**, then mints a token that lets the anonymous render page read just that
one audit's report. The token is an HMAC over `auditId` + an expiry, compared in
constant time (`lib/crypto.ts:78-95`):

```ts
// lib/crypto.ts:78-81
export function signAuditPdfToken(auditId: string, ttlMs = PDF_TOKEN_TTL_MS): string {
  const exp = Date.now() + ttlMs;
  return `${exp}.${tokenHmac(`${auditId}.${exp}`)}`;
}
```

### The legacy react-pdf fallback

For audits created **before** migration `0063` there is no `reportJson`, so the
route falls back to the original lightweight document. It loads the issue rows
and renders `SpyroAuditPdf` (from `lib/audit/pdf-report.tsx`) straight to a
buffer with `renderToBuffer` - no browser, no second HTTP hop
(`route.ts:83-117`):

```ts
// app/api/audit/[auditId]/pdf/route.ts:96-108
const logoSrc = getLogoBase64(); // logo memoised as a base64 data URI at module scope
const pdfBuffer = await renderToBuffer(
  createElement(SpyroAuditPdf, {
    audit: { startedAt, seoScore, geoScore, pagesCrawled },
    issues: issueRows as PdfIssueData[],
    domain,
    logoSrc,
  }) as ReactElement<DocumentProps, …>,
);
```

`SpyroAuditPdf` is a server-only react-pdf component (`Document`/`Page`/`View`/
`Text`/`Image` + a `StyleSheet`). It uses the **built-in Helvetica faces** only
(no web fonts) and hard-coded hex colours (react-pdf has no CSS variables), and
parses GEO `**bold**` fix-text manually via `PdfFixText`
(`lib/audit/pdf-report.tsx:22-73`). The buffer is returned with
`Content-Disposition: attachment` and `Cache-Control: private, no-store`
(`route.ts:110-117`).

## The free-tool download routes

Each free tool exposes a small `GET` route under `app/api/<tool>/pdf/route.ts`
that validates input, checks the report exists and is fresh, then delegates to
`renderReportPdf`. The free-audit route is representative:

```ts
// app/api/free-audit/pdf/route.ts:6-36
export const runtime = "nodejs";
export const maxDuration = 300; // the Vercel function just waits on Gotenberg

export async function GET(req: NextRequest) {
  const website = new URL(req.url).searchParams.get("website");
  if (!website) return new NextResponse("Missing website parameter", { status: 400 });
  const norm = normalizeDomainInput(website);
  if (!norm) return new NextResponse("Invalid website", { status: 400 });

  const row = await getReportRow(norm.domain).catch(() => null);
  if (!row || !isFresh(row)) {
    return new NextResponse("Report not found or expired. Please run the audit again.", { status: 404 });
  }
  return renderReportPdf({
    req,
    tool: "free-audit",
    reportPath: "/pdf-report",
    query: `website=${encodeURIComponent(norm.domain)}`,
    filename: `Spyro_Audit_${norm.domain}.pdf`,
  });
}
```

`renderReportPdf` now has **six** consumers: the five free-tool routes
(`app/api/{ai-crawler-check,ai-visibility,free-audit,meta-snippet,schema-validator}/pdf/route.ts`,
each with its own `reportPath` and filename) plus the authed audit route, which
passes `tool: "audit"` and `reportPath: "/audit-report"`.

### Client download

In the browser, the **Download PDF** button does not `window.open` - it fetches
the route as a blob and triggers a real file download with the right filename,
via the shared `downloadFreeToolPdf` helper:

```ts
// lib/free-tools/shared/download-pdf.ts:11-29
export async function downloadFreeToolPdf(pdfUrl: string, filename: string): Promise<PdfDownloadResult> {
  const res = await fetch(pdfUrl, { method: "GET", headers: { Accept: "application/pdf" } });
  if (!res.ok) { … return { ok: false, error: `HTTP ${res.status}` }; }
  const blob = await res.blob();
  const a = document.createElement("a");
  a.href = URL.createObjectURL(blob);
  a.download = filename;
  a.click();
  // … cleanup …
  return { ok: true };
}
```

Returning `{ ok }` lets the button show a retry state when generation fails.

## Generate → download flow

```mermaid
sequenceDiagram
  participant U as Browser
  participant R as /api/audit/[id]/pdf (Vercel)
  participant DB as audits
  participant G as Gotenberg (Railway, Chromium)
  participant P as /audit-report page (token-gated)

  U->>R: GET /api/audit/{id}/pdf
  R->>DB: audit scoped to user.id
  alt not owner / not done
    R-->>U: 401 / 404
  else owns a modern audit (reportJson set)
    R->>R: signAuditPdfToken(auditId)
    R->>G: POST /forms/chromium/convert/url (audit-report URL + token)
    G->>P: GET https://origin/audit-report?auditId=…&token=…
    P->>P: verifyAuditPdfToken → ok
    P->>DB: read audits.report_json
    P-->>G: <AuditReportDocument variant="app"/> HTML + CSS
    G-->>R: application/pdf bytes
    R-->>U: PDF (attachment; no-store)
  else legacy audit (no reportJson)
    R->>DB: load issue rows
    R->>R: renderToBuffer(SpyroAuditPdf)
    R-->>U: PDF (attachment; no-store)
  end
```

The free-tool flow is the same Gotenberg path, minus the token: the route checks
`getReportRow` + `isFresh`, then renders `/pdf-report?website=…` (public) through
Gotenberg.

## A note on `@resvg/resvg-js`

`@resvg/resvg-js` sits beside `@react-pdf/renderer` in `serverExternalPackages`
(`next.config.ts:37`), so it is easy to assume it is part of PDF generation. It
is **not**. Its only consumer is the **content/image pipeline**:
`lib/images/chart.ts` hand-rolls a grouped-bar SVG and rasterises it to PNG with
`Resvg`, used by the blog in-content chart generator. Neither PDF path embeds a
resvg chart - the shared document draws its gauges as CSS `conic-gradient` divs.
See [Content engine](/backend/content-engine).

## Common mistakes and tips

<AccordionGroup>
<Accordion title="PDF 404s in local dev">
Gotenberg can't fetch `http://localhost:3000`. Set `GOTENBERG_REPORT_ORIGIN` to
a public origin that serves the same database (`browser.ts:35-41`), or the route
returns *"No public origin for the report page."*
</Accordion>
<Accordion title="The in-app audit export 404s the render page">
`/audit-report` requires a valid, unexpired HMAC token minted by the PDF route
(`signAuditPdfToken`). The token has a short TTL (`PDF_TOKEN_TTL_MS`,
`lib/crypto.ts:78`) - if Gotenberg is slow or the clock skews past expiry, the
page 404s. The route only mints a token **after** the auth + ownership check.
</Accordion>
<Accordion title="The PDF is blank / fonts look wrong">
The Gotenberg render waits `1s` (`waitDelay`) for Inter/Lato to settle
(`browser.ts:101`). The legacy react-pdf fallback can only use the built-in
Helvetica faces - do not reference a web font there (`pdf-report.tsx:64-73`).
</Accordion>
<Accordion title="Changing PDF layout">
For both the free tools and the modern audit, edit the **shared document**
(`components/audit/audit-report-document.tsx`, real Tailwind/CSS) - that one file
changes both PDFs. The legacy audit fallback is the separate
`lib/audit/pdf-report.tsx` (react-pdf primitives only, no arbitrary CSS) and only
affects pre-`0063` audits.
</Accordion>
</AccordionGroup>
