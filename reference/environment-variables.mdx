This is the authoritative list of every environment variable Spyro reads, built
from `.env.local.example` and cross-checked against `lib/env.ts`. It documents
**names and purposes only** ‚Äî never copy a real secret value here, and never open
`.env.local`.

<Info>
**How Spyro reads env.** `lib/env.ts` reads each key with a light-touch `opt()`
helper (empty string ‚áí `undefined`) and **does not throw at import time** for most
keys ‚Äî features degrade to mocks when a key is absent. Only `NEXT_PUBLIC_APP_URL` is
effectively required (it throws when a consumer needs the public origin and no
Vercel fallback exists) *(`lib/env.ts:1‚Äì11,38‚Äì59`)*. Feature on/off is decided by the
`featureFlags` object *(`lib/env.ts:189‚Äì222`)*.
</Info>

<Note>
**Coverage:** ~70 variables ship in `.env.local.example`. A few more are **read in
code but not in the example file** ‚Äî they are listed in their sections and marked
*(code-only)*. "Default" columns cite values supplied by `lib/env.ts` even when the
example leaves the line commented.
</Note>

## Supabase (auth + database)

| Variable | Required | Purpose / enables |
| --- | --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` | **Required** | Supabase project URL. Auth + the whole dashboard. |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | One of these two | New-style `sb_publishable_‚Ä¶` key (**preferred**). |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | One of these two | Legacy anon JWT (fallback if publishable unset). Whichever is set wins *(`lib/env.ts:65‚Äì66`)*. |
| `SUPABASE_SERVICE_ROLE_KEY` | **Required** | Server-side service role (bypasses RLS; also default encryption-key fallback and Storage uploads). |
| `DATABASE_URL` | **Required** | Postgres connection string used by Drizzle (`drizzle.config.ts`). |
| `NEXT_PUBLIC_TURNSTILE_SITE_KEY` | Optional | Cloudflare Turnstile **site** key ‚Äî bot/spam widget on auth forms. The secret goes in Supabase, not env. |

See [Authentication](/backend/authentication) and [Database](/backend/database).

## LLM ‚Äî OpenRouter (primary chat provider)

One key routes every chat model: blog writer, ideation, the Spyro agent, the
citation engines, and image generation.

| Variable | Required | Purpose / enables |
| --- | --- | --- |
| `OPENROUTER_API_KEY` | **Required** for real LLM output | Routes all chat LLM calls + Seedream image gen. |
| `OPENROUTER_PREMIUM_MODEL` | Optional | Top-quality model (topic ideas, titles, outline + meta). Default `anthropic/claude-sonnet-4.6`. |
| `OPENROUTER_CAPABLE_MODEL` | Optional | Bulk blog section drafting. Default `moonshotai/kimi-k2.6`. |
| `OPENROUTER_FAST_MODEL` | Optional | Parsing / classification. Default `openai/gpt-4.1-mini`. |
| `OPENROUTER_AGENT_MODEL` | Optional | The Spyro chat agent. Default `moonshotai/kimi-k2.6`. |
| `OPENROUTER_IMAGE_MODEL` | Optional | Featured + in-content images. Default `bytedance-seed/seedream-4.5`. |
| `OPENROUTER_IMAGE_SIZE` | Optional | Image output size. Left unset (Seedream is flat-priced). |
| `AUDIT_ASSISTANT_MODEL` | Optional | Ask-Spyro audit sidebar model. Default `minimax/minimax-m2.7`; GPT-4.1-mini is the runtime fallback *(`lib/env.ts:85`)*. |

See [AI providers](/backend/ai).

## OpenAI (embeddings + LLM fallback)

| Variable | Required | Purpose / enables |
| --- | --- | --- |
| `OPENAI_API_KEY` | Recommended | RAG embeddings (`text-embedding-3` family ‚Äî the site-index / vector search). Also a tier-3 chat + image fallback when OpenRouter is unset. |

<Note>
The source disagrees on the embedding model: `.env.local.example` says
`text-embedding-3-large @ 1536`, `lib/env.ts:87` says `text-embedding-3-small`.
Confirm in `lib/embed` before relying on a dimension. See [AI providers](/backend/ai).
</Note>

## Rerank (Cohere via OpenRouter)

`OPENROUTER_API_KEY`. All optional; `lib/env.ts` supplies defaults.

| Variable | Required | Purpose / enables (default) |
| --- | --- | --- |
| `RERANK_ENABLED` | Optional | On/off. Default `true` *(`lib/env.ts:142`)*. |
| `RERANK_PROVIDER` | Optional | Default `openrouter`. |
| `RERANK_MODEL` | Optional | Default `cohere/rerank-4-fast`. |
| `RERANK_TOP_N` | Optional | Chunks passed to the LLM. Default `8`. |
| `RERANK_TIMEOUT_MS` | Optional | Rerank request timeout. Default `4000`. |
| `VECTOR_CANDIDATE_K` | Optional | pgvector candidates fed to the reranker. Default `24`. |

## Superadmin panel ‚Äî dedicated credentials

All env-only (never DB columns). The panel **404s entirely** unless `ADMIN_USERNAME`
+ a password + `ADMIN_SESSION_SECRET` + `ADMIN_TOTP_SECRET` are all set
(`adminConfigured`, `lib/env.ts:216‚Äì221`).

| Variable | Required | Purpose / enables |
| --- | --- | --- |
| `ADMIN_USERNAME` | Required for panel | Superadmin login username. |
| `ADMIN_PASSWORD_HASH` | One of these two | `scrypt:salt:hash` of the password (never plaintext). |
| `ADMIN_PASSWORD` | One of these two | Plaintext password used directly; wins if both set. |
| `ADMIN_SESSION_SECRET` | Required for panel | HMAC secret for the 1-hour admin session cookie. |
| `ADMIN_TOTP_SECRET` | Required for panel | base32 2FA secret (scan into an authenticator). |
| `ADMIN_IP_ALLOWLIST` | Optional | Comma-separated IPs / IPv4 CIDRs allowed to reach `/superadmin`. Empty = no IP limit. |

See [Middleware](/backend/middleware) and [Security](/backend/security).
